Detailed Description

This class represents device components for which a corresponding CAN command is existing.

The CAN ID is built from the controller ID and the unit number of this Device.

See also
Controller::id()

enum class mrw::model::Device::LockState : int
strong

This enumeration represents the processing state of this Device.

Enumerator
FAIL 

An error occured during pending processing.

UNLOCKED 

The Device is unlocked and can be modified.

PENDING 

The Device is actually modifying its state.

LOCKED 

The Device is in a state where state modification is prohibited.

◆ isUnlockable()

virtual bool mrw::model::Device::isUnlockable ( ) const
pure virtual

This method returns true if it is possible to unlock this Device.

There are some conditions to be met to ensure a safe unlock. This method is called during lock() method with a given parameter of LockState::UNLOCKED.

Note
This test method is meant that a locked Device is unlockable. It is not meant that if this Device is unlocked and not lockable!
Returns
True if this Device is safely possible to unlock.

◆ unitNo()

UnitNo Device::unitNo ( ) const

This method returns the unit number which is part of the CAN identifier sent to the controller with a CAN command.

It is directly used inside the CAN extended frame format ID.

Note
Even the extended frame format ID is 18 bits wide only 16 bits are used from this unit number.
Returns
The unit number of this Device.
