Detailed Description

This class represents all Signal variants.

It may represent LightSignal parts as well as FormSignal parts.

Note
In fact a Section can only have six Signal parts maximum: Every SignalType in both directions.

These symbols appears on the trck control application:

  1. Shunting Signal:
    1. Shunting stop:
    2. Shunting go:
  2. Distant Signal:
    1. Distant expect stop:
    2. Distant expect go:
  3. Main Signal:
    1. Main stop:
    2. Main outgoing stop:
    3. Shunting outgoing go:
  4. Main Signal combined with distant Signal:
    1. Tour go:
    2. Tour go expect stop:
    3. Tour go expect go:

enum class mrw::model::Signal::Symbol : int

The strategic Symbol of that Signal instance to display/show.

This will be computed into a mrw::can::SignalState later depending on the SignalType.

Enumerator
OFF 

Will always result in SIGNAL_OFF.

STOP 

Will result in SIGNAL_HP0, SIGNAL_SH0 or SIGNAL_VR0.

GO 

Will result in SIGNAL_HP1, SIGNAL_SH1 or SIGNAL_VR1.

In case that at least two curved RailPart elements occur between two main signals a SIGNAL_VR2 or SIGNAL_HP2 may occur.

Note
The ESTW does not distinguish between SIGNL_VR1 or SIGNAL_VR2 and SIGNAL_HP1 or SIGNAL_HP2 respectively.

◆ less() [2/2]

bool Signal::less ( Signal * left,
Signal * right )
staticnodiscard

This comparator method returns true if the "left" Signal has higher priority than the "right" Signal.

Left or right is not a topology but a mathmatical meaning. It means left from and right from "<" sign. There are two criteria to sort:

  1. The orientation related to counting direction
  2. The SignalType where a main Signal has higher priority than a distant Signal and a shunting Signal.
Parameters
leftThe left of "<" Signal.
rightThe right of "<" Signal.
Returns
True if left has higher priority than right.

◆ parse() [2/2]

void Position::parse ( QSettings & settings,
const QString & default_value )
inherited

This method reads a value from the given mrw::util::Settings using the key() method as key and in case the value is not available it uses the default_value parameter instead.

The read value is parsed as numerical coordinates. If the coordinates are not parseable a default post increment counter is used to locate an alternative position. The coordinates may extended by codes to produce a half shift in position and/or multiple half extends in size.

Note
The Settings use the "positions" group for storage.

Example: 12,5,ooxxx leading in one offset shift and additional 3 half extends in size.

Parameters
settingsThe settings to use.
default_valueThe default value to use.
